Sugiro pensar em duas tarefas separadas e integra-las depois:
1. Você pode fazer um sistema de cadastro de pacientes online usando Flask. Inclusive, se uma interface bonita ou mais intuitiva não for um requisito neste momento inicial (por exemplo, se só você for testar o sistema agora), você pode usar o próprio painel do Flask-Admin: https://flask-admin.readthedocs.io/en/latest/

2. Supondo que você tem os dados que precisa, você pode preencher um documento com eles. Uma opção é continuar com o estilo web e formatar uma página HTML com esses dados. Se for usar Flask, Django ou outra abordagem web no passo 1, faz sentido manter o documento como web também. Outra opção seria construir um documento tipo Word, marcar pontos em que os dados seriam substituídos e automatizar essa substituição em novos arquivos com dados recuperados dos pacientes.

Um detalhe para se atentar: não conhecemos  as legislações que regem dados de pacientes. É importante inteirar-se disso, porque a lei pode especificar requisitos para o armazenamento correto destes dados em bancos de dados.
